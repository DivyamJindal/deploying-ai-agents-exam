02 · Architecture choice

Why LangGraph, not a single chatbot.

Single chatbot

  • Conditional routing buried in a prompt
  • No clean way to pause for a human
  • State is implicit, untyped, easily lost
  • "Resume" means replaying the full chat
  • Sentiment + escalation rules drift together

LangGraph

  • Explicit nodes for classify / solve / monitor / escalate
  • interrupt() pauses and persists waiting for the human
  • SqliteSaver checkpoints survive process restarts
  • Command(resume=…) picks up at the exact node
  • Pydantic SupportState = one schema, one source of truth

key insight   The interesting work isn't the LLM call. It's the orchestration around it.

04 · State

One Pydantic schema, end to end.

class SupportState(BaseModel):
    ticket: TicketInput
    category: TicketCategory | None = None
    resolution_attempts: list[ResolutionAttempt] = []
    sentiment_score: float = 0.0
    escalation_reason: str | None = None
    supervisor_input: str | None = None
    final_response: str | None = None
    ticket_metadata: dict[str, str] = {}

Every node returns a partial state dict. LangGraph merges. Pydantic validates. The UI never sees a raw __interrupt__ tuple it can't render.

The normalization layer

Streamlit can receive a Pydantic model, a dict, or a LangGraph Interrupt. ui_state.to_plain() recursively converts anything into JSON-safe primitives. 

def to_plain(obj):
    if isinstance(obj, BaseModel):
        return to_plain(obj.model_dump())
    if isinstance(obj, dict):
        return {k: to_plain(v) for k, v in obj.items()}
    if isinstance(obj, (list, tuple)):
        return [to_plain(x) for x in obj]
    return obj

Killed the 'ResolutionAttempt' object is not subscriptable bug for good. 17 / 17 tests

09 · Edge cases

What the graph handles by design.

Missing account

Solver attempt records the gap, gate inspects, supervisor decides whether to ask the customer or proceed.

Unresolved bug

Solver returns resolved=False. Gate trips. Supervisor sees the diagnosis trail.

Angry tone

Sentiment ≥ 0.67 always escalates regardless of category — the cheapest CSAT save we own.

Refund > threshold

Hard-coded $200 cap — adjustable via supervisor guidance, never auto-fired by the LLM.

Supervisor rejection

Final response includes the rejection guidance and stays open; doesn't auto-resolve.

LLM failure

Classifier and sentiment fall back to deterministic rule-based scoring. Demo never breaks.

10 · Learnings & next

What we'd ship next.

Learnings

  • Routing + interrupts > one big prompt
  • Pydantic + a normalization layer = bug-free UI
  • Persistent checkpoints turn the demo into a real product
  • LLM is a node, not the whole agent

Next steps

  • SLA timers — sidebar control already wired
  • Zendesk webhook adapter (one-way ingest first)
  • Multilingual response drafting
  • CSV / PDF export from the Inbox
  • RAG into the KB for solver grounding

The whole thing runs in <200ms without an LLM, and ~2s with gpt-4.1-nano. The supervisor is the long pole — and that's correct.
